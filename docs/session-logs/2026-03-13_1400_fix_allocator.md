# fix(allocator): remove hardcoded strategy_type fallback, enforce explicit strategy validation

## What was done

Fixed the root cause of "float() argument must be a string or a real number, not 'NoneType'" errors in recursive_lending rebalance calculations. The bug occurred because positions were deployed via the allocator dashboard with `strategy_type='recursive_lending'` when they were actually different strategy types (perp_lending, etc.). The allocator had a hardcoded fallback to `'recursive_lending'` for unrecognized strategy types — a legacy from before perp strategies were added.

**Root cause chain:**
1. Dashboard allocator (built pre-perp-strategies) had `strategy.get('strategy_type', 'recursive_lending')` fallback
2. Positions deployed as perp_lending/other but stored as 'recursive_lending'
3. Rebalance calculator selected recursive_lending calculator (wrong one)
4. Recursive_lending calculator expects `entry_token3_amount` to never be NULL for L_B leg
5. Perp_lending positions have `entry_token3_amount=NULL` (unused leg)
6. Float conversion crashes: `float(None)`

**Fixes applied:**
- Removed hardcoded 'recursive_lending' fallback in dashboard_renderer.py (lines 902, 996)
- Changed `strategy.get('strategy_type', default)` to `strategy['strategy_type']` (fail loud if missing)
- Added explicit ValueError in recursive_lending.py when entry_token3_amount is NULL (Design Note #13)
- Fixed iterative liquidity updates: NaN vs 0.0 distinction (NaN = no data, 0.0 = depleted)
- Removed `.get()` defaults for token2/token4_available_borrow in matrix population (Design Note #16)

## Decisions made

- **Explicit strategy_type validation**: Use direct dictionary access instead of .get() with defaults. This forces strategy_type to be present, preventing silent misassignments.
- **Fail loudly on data problems**: Added ValueError when entry_token3_amount is NULL for recursive_lending, allowing immediate identification of positions with wrong strategy_type.
- **NaN vs 0.0 distinction**: Changed matrix initialization from `0.0` to `float('nan')` to distinguish "no liquidity data available" from "liquidity pool depleted". This prevents accidental zero-based calculations.
- **Token naming alignment**: Updated field names from `available_borrow_3b` to `token4_available_borrow` to match canonical token1/2/3/4 naming (Design Note #18).
- **Deferred**: Strategy type constants still not extracted to settings.py (can be done in future if needed; currently works well with direct field access).
